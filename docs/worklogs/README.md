## Purpose

This directory records Httpora maintenance work, architectural decisions,
governance evidence, and follow-up items that should remain discoverable inside
the repository.

## Ledgers

| File | Purpose |
| --- | --- |
| [`WORK_LOG.md`](WORK_LOG.md) | Chronological maintenance and validation log |

## Entry Format

Each entry should include:
- **Date** in `YYYY-MM-DD` format
- **Category** such as `GOVERNANCE`, `DEPENDENCIES`, `ARCHITECTURE`, or `TESTING`
- **Context** describing why the work happened
- **Decision/Finding** describing what changed or was learned
- **Validation** listing commands or external checks
- **Tags** for cross-repo aggregation
